RESUMEN DE MODIFICACIONES REALIZADAS:

1. **VISTA PÚBLICA DE HISTORIAL QR (vehicle_history_qr.php)**:
   - Ya era una vista pública independiente (no hereda layout del sistema)
   - Agregado botón "NUEVA ORDEN" para administradores
   - El botón aparece solo si: 
     * Usuario está logueado (`$_SESSION['user_id']`)
     * Usuario es administrador (`$_SESSION['user_role'] === 'ADMINISTRADOR'`)
   - El botón redirige a: `/taller/nueva_orden?placa=PLACA_DEL_VEHICULO`

2. **CONTROLADOR DE TALLER (ControllerTaller.php)**:
   - Modificado método `nuevaOrden()` para aceptar parámetro GET `?placa=ABC123`
   - Auto-busca vehículo por placa y carga sus datos
   - Busca información del cliente asociado al vehículo

3. **VISTA DE NUEVA ORDEN (nueva_orden.php)**:
   - Agregado mensaje visual cuando los datos son pre-cargados
   - Campos auto-llenados con datos del vehículo:
     * Placa
     * ID Cliente
     * Nombre del Cliente
     * Marca
     * Modelo
     * Año
     * Color
   - Estilos visuales (fondo verde) para indicar campos pre-cargados
   - Lógica JavaScript para buscar automáticamente el último kilometraje

4. **FUNCIONALIDAD COMPLETA**:
   - Usuario normal/visitante: Ve historial limpio sin botones administrativos
   - Usuario administrador logueado: Ve historial + botón "NUEVA ORDEN"
   - Al hacer click en "NUEVA ORDEN": Se redirige al formulario con datos del vehículo pre-cargados
   - El formulario de nueva orden ya incluye el último kilometraje del vehículo

5. **RUTAS Y ACCESO**:
   - Historial público: `/consultas/showVehicleHistoryByQr/{placa}`
   - Nueva orden con datos pre-cargados: `/taller/nueva_orden?placa={placa}`

¿CÓMO PROBAR?
1. Generar QR para una placa existente
2. Escanear QR desde dispositivo móvil o navegador no logueado → Ver historial limpio
3. Escanear QR desde navegador logueado como administrador → Ver historial + botón "NUEVA ORDEN"
4. Hacer click en "NUEVA ORDEN" → Ver formulario con datos del vehículo pre-cargados
